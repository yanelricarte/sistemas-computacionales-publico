Mini-sistema web con login

Transformar el login básico que veníamos usando en un mini-sistema funcional: CSS común, redirecciones, sesiones, páginas protegidas y navegación con GET. Sin base de datos todavía.

Materia: Proyecto, Diseño e Implementación de Sistemas Computacionales · 7.º año
Tiempo de trabajo en clase: 50 minutos

Consigna

Tomar el login básico (usuario + contraseña) que vienen trabajando y transformarlo en un mini-sistema web funcional con:

  • CSS común en archivo aparte.
  • Redirecciones server-side después de validar.
  • Sesiones que recuerden al usuario entre páginas.
  • Páginas internas protegidas (no se entra escribiendo la URL directa).
  • Navegación con GET para una página de información.
Sin base de datos todavía
Los usuarios están hardcodeados en el código (escritos directamente). La autenticación real, con contraseñas hasheadas en base de datos, la vemos cuando entremos a MySQL.

Obligatorio (piso del TP)

1. Estructura de carpetas

clase02/
├── index.php              ← formulario de login
├── procesar-login.php     ← valida, abre sesión, redirige
├── bienvenido.php         ← página interna protegida
├── info.php               ← secciones vía GET
├── logout.php             ← destruye sesión
├── includes/
│   ├── cabecera.php       ← <head>, <link CSS>, navbar
│   └── pie.php
└── css/
    └── estilos.css        ← todos los estilos

2. index.php — Formulario de login

  • Inputs: usuario (texto), clave (password).
  • Envía por POST a procesar-login.php.
  • Botón "Entrar".

3. procesar-login.php — Validación + sesión + redirect

  • session_start() al principio.
  • Validar contra al menos un usuario hardcodeado (admin/1234 está bien).
  • Si OK: guardar $_SESSION['usuario'], redirigir a bienvenido.php.
  • Si MAL: redirigir a index.php.
  • Siempre header('Location: ...'); seguido de exit;.

4. bienvenido.php — Página interna protegida

  • session_start() al principio.
  • Si $_SESSION['usuario'] no existe, redirigir a index.php con exit;.
  • Mostrar "Hola, [nombre del usuario]" usando el dato de la sesión.
  • Incluir cabecera.php y pie.php.

5. info.php — Secciones dinámicas con GET

  • Protegida por sesión (mismo patrón que bienvenido.php).
  • Leer $_GET['seccion'] con default razonable (?? 'ayuda').
  • Mostrar contenido distinto según el valor: ayuda, contacto, nosotros.

6. logout.php — Cerrar sesión

  • session_start(), session_destroy(), redirigir a index.php.

7. cabecera.php y pie.php — Comunes

  • cabecera.php: <head> con <link rel="stylesheet" href="css/estilos.css">, abrir <body>, navbar con links a Inicio, Ayuda, Contacto, Salir.
  • pie.php: cerrar <body> y <html>.

8. css/estilos.css — CSS común

  • Todos los estilos en este único archivo, linkeado desde cabecera.php.
  • Selectores de tipo (body, nav, h1, form) para estilos base.
  • Selectores de clase (.boton, .mensaje, etc.) para variantes.
  • Si tenías style="..." inline en archivos previos, movelos a estilos.css como reglas.

Opcional (si terminás antes)

  • Más usuarios: aceptar admin/1234, prof/abc, alumno/2026.
  • Mensajes distintos: que el login distinga entre "no completaste un campo" y "usuario o contraseña incorrectos". Se puede pasar el motivo por GET de vuelta a index.php (?error=vacio o ?error=invalido) o usar sesión.
  • Saludo personalizado en bienvenido.php: mostrar también la hora actual o un mensaje según el rol del usuario.
  • Más secciones en info.php: agregar ?seccion=ayuda-php, ?seccion=git, etc.
  • Estilos con clases: aplicar .boton, .mensaje, .tarjeta con variantes (ver el material teórico).

Paso a paso para implementarlo

No intentes armarlo todo de una. Andá de a un paso por vez, probando que funcione antes de pasar al siguiente. Si te trabás en alguno, mirá el material teórico de la sección correspondiente o pedile ayuda a la docente.

El código de esta guía es un ejemplo — adaptalo a tu sistema

Los bloques de código que aparecen en cada paso son una versión mínima de referencia, no el código que tenés que copiar tal cual. Tu sistema puede tener variaciones: otros usuarios válidos, más secciones en info.php, distintos estilos, mensajes propios. Lo que importa es entender qué hace cada parte y por qué va en ese orden, no transcribirlo letra por letra.

Si copiás sin entender, cuando algo no funcione no vas a poder arreglarlo — y eso te lo van a notar enseguida.

Antes de empezar
Creá la carpeta clase02/ donde guardás tus archivos del año. Abrila en VS Code con Archivo → Abrir carpeta... (no archivo por archivo). Eso te ahorra muchos problemas con rutas.

Paso 1 — Estructura mínima de carpetas

Antes de escribir código, armá la estructura vacía:

clase02/
├── includes/        (carpeta vacía)
└── css/             (carpeta vacía)

Solo las carpetas, sin archivos todavía. Esto te obliga a pensar la organización antes de empezar.

Paso 2 — Cabecera, pie y CSS común

Antes que el login, armá el "esqueleto" del sitio. Tres archivos:

  1. includes/cabecera.php con la apertura del HTML, <head>, el <link> al CSS y la navbar (de momento, links que no apuntan a nada todavía): 
    <!DOCTYPE html>
<html lang="es">
<head>
  <meta charset="UTF-8">
  <title>Mi sistema</title>
  <link rel="stylesheet" href="css/estilos.css">
</head>
<body>
  <nav>
    <a href="index.php">Inicio</a>
    <a href="info.php?seccion=ayuda">Ayuda</a>
    <a href="logout.php">Salir</a>
  </nav>
  2. includes/pie.php con el cierre: 
    </body>
</html>
  3. css/estilos.css vacío por ahora (lo vas llenando a medida que avanzás).

Paso 3 — Página de inicio (formulario de login)

Creá index.php que incluya cabecera y pie, y tenga el formulario:

<?php include 'includes/cabecera.php'; ?>

<h1>Iniciar sesión</h1>

<form action="procesar-login.php" method="POST">
  <label>Usuario:
    <input type="text" name="usuario">
  </label>
  <label>Contraseña:
    <input type="password" name="clave">
  </label>
  <button type="submit">Entrar</button>
</form>

<?php include 'includes/pie.php'; ?>

Probalo en el navegador. Tiene que verse el formulario. Apretá Entrar y va a tirar error porque procesar-login.php no existe todavía. Está bien, próximo paso.

Paso 4 — Procesar el login (sin sesión todavía)

Creá procesar-login.php. Solo validación + redirect, sin sesión por ahora:

<?php
  $usuario = $_POST['usuario'] ?? '';
  $clave   = $_POST['clave']   ?? '';

  if ($usuario == 'admin' && $clave == '1234') {
    echo "Login OK (después acá redirigimos)";
  } else {
    echo "Login MAL (después acá volvemos a index)";
  }

Probalo. Cargá el formulario, escribí admin / 1234 → tiene que decir "Login OK". Probá con otros datos → "Login MAL". Si esto no funciona, todavía no tiene sentido seguir.

Paso 5 — Sumar sesión y redirect

Ahora reemplazá los echo por los redirects reales y guardá el usuario en la sesión:

<?php
  session_start();

  $usuario = $_POST['usuario'] ?? '';
  $clave   = $_POST['clave']   ?? '';

  if ($usuario == 'admin' && $clave == '1234') {
    $_SESSION['usuario'] = $usuario;
    header('Location: bienvenido.php');
    exit;
  } else {
    header('Location: index.php');
    exit;
  }

Importante: session_start() tiene que ir antes de cualquier echo o HTML (en este archivo no hay HTML, perfecto).

Paso 6 — Página interna protegida (bienvenido.php)

Creá bienvenido.php con el patrón de protección al inicio:

<?php
  session_start();
  if (!isset($_SESSION['usuario'])) {
    header('Location: index.php');
    exit;
  }
?>
<?php include 'includes/cabecera.php'; ?>

<h1>Hola, <?= $_SESSION['usuario'] ?></h1>
<p>Estás dentro del sistema.</p>

<?php include 'includes/pie.php'; ?>

Probalo. Logueate desde index.php → tiene que llevarte a bienvenido y mostrar "Hola, admin". Ahora probá la protección: cerrá el navegador (o abrí incógnito), escribí directo la URL de bienvenido.php → tiene que mandarte al login.

Paso 7 — Logout

Creá logout.php para cerrar sesión:

<?php
  session_start();
  session_destroy();
  header('Location: index.php');
  exit;

Probalo. Logueate, andá a bienvenido, después clickeá "Salir" en la navbar (que apunta a logout.php) → tiene que devolverte al login. Probá ir a bienvenido directo → te bota al login (porque ya no tenés sesión).

Paso 8 — Página con secciones dinámicas (GET)

Creá info.php protegida y con contenido según ?seccion=...:

<?php
  session_start();
  if (!isset($_SESSION['usuario'])) {
    header('Location: index.php');
    exit;
  }
  $seccion = $_GET['seccion'] ?? 'ayuda';
?>
<?php include 'includes/cabecera.php'; ?>

<?php if ($seccion == 'ayuda'): ?>
  <h1>Ayuda</h1>
  <p>Esto es un sistema de ejemplo. Logueate con admin / 1234.</p>
<?php elseif ($seccion == 'contacto'): ?>
  <h1>Contacto</h1>
  <p>Escribinos a contacto@ejemplo.com</p>
<?php elseif ($seccion == 'nosotros'): ?>
  <h1>Nosotros</h1>
  <p>Somos un equipo de 7° año aprendiendo PHP.</p>
<?php endif; ?>

<?php include 'includes/pie.php'; ?>

Acordate de tener los links a info.php?seccion=... en la navbar de cabecera.php.

Paso 9 — Estilos en estilos.css

Hasta acá el sitio funciona pero se ve "feo" (HTML sin estilo). Es momento de meterle CSS:

  • Estilos base: body, nav, h1, form, input, button.
  • Si tenés clases (.boton, .mensaje, etc.), agregalas en HTML y en CSS.
  • Si tenías style="..." inline en archivos previos, movelos a estilos.css.

Paso 10 — Probá todo el flujo entero

  1. Abrir navegador en modo incógnito (sesión limpia).
  2. Ir a http://localhost/clase02/bienvenido.php directo → te debería echar al login.
  3. Loguearte con admin/1234 → te debería llevar a bienvenido.
  4. Clickear los links de la navbar (Ayuda, Contacto, Nosotros) → cada uno carga su contenido.
  5. Logout → vuelve a login.
  6. Probar la URL directa de bienvenido otra vez → te echa de nuevo.

Si los 6 puntos funcionan, tenés el TP terminado. Si querés sumar techo (opcionales), seguí con la lista de arriba.

Si te trabás
  • "Headers already sent": revisá que el <?php sea la primera línea del archivo. Sin espacios ni líneas en blanco antes.
  • La sesión "se pierde" entre páginas: te falta session_start() en alguna página.
  • CSS no carga: revisá la ruta del <link> con F12 → Network.
  • Ver errores frecuentes en la teoría para más casos.

Reglas del TP

  • Código: trabajo individual, cada uno en su PC. Cada uno entrega su carpeta clase02/.
  • Proyecto anual: pueden conversar en grupo mientras programan (viabilidad, usuarios, qué resuelve cada idea).
  • Pueden consultar la documentación oficial de PHP.
  • Pueden consultar al compañero sobre código, pero no copiar/pegar sin entender — la solución que no podés explicar no cuenta.
