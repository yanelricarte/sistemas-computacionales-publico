1. ¿De qué se trata esta materia?

El nombre completo no es decorativo. Cada palabra significa algo.

  • Proyecto. Durante el año vamos a construir una sola cosa, sostenida en el tiempo, con principio y final. No son ejercicios sueltos: es un trabajo que arrancamos en mayo y entregamos en noviembre.
  • Diseño. Antes de programar, vamos a pensar. Qué hace el sistema, quién lo usa, qué datos guarda, qué reglas tiene. La parte más difícil de programar no es escribir código: es decidir qué código hay que escribir.
  • Implementación. No alcanza con que esté pensado en una carpeta o en un PowerPoint. Tiene que funcionar.

Programar es escribir instrucciones para una computadora.
Construir un sistema es resolver el problema real de alguien usando esas instrucciones.
Son dos cosas distintas.

2. ¿Qué es un sistema informático?

Un sistema informático es un conjunto de cinco partes que trabajan juntas:

              USUARIO
                 |
                 v
   ENTRADA  →  PROCESO  →  SALIDA
                 |
                 v
             ALMACEN

5. GET vs POST

GET — los datos van en la URL 
https://www.google.com/search?q=php+formularios
                              ↑
                              datos
  • Visibles en la URL.
  • Se pueden compartir como link.
  • Quedan en el historial.
  • Sirve para: búsquedas, filtros, paginación.
  • No sirve para: contraseñas, datos privados.
POST — los datos van ocultos
  • No quedan en la URL.
  • No quedan en el historial.
  • Sirve para: enviar formularios, guardar datos, contraseñas.
  • Es lo que vamos a usar siempre este año.
Resumen
GET = buscar. POST = enviar. Hoy todos usamos POST.

6. Estructura PHP profesional 

clase01/
├── index.php          ← formulario
├── procesar.php       ← recibe y procesa
├── includes/
│   ├── cabecera.php   ← <head>, navbar, apertura de <body>
│   └── pie.php        ← cierre, scripts de Bootstrap
└── css/
    └── estilos.css    ← estilos propios

¿Por qué separar cabecera.php y pie.php?

Porque tanto index.php como procesar.php van a tener el mismo <head>, navbar, footer. Si los escribimos dos veces, cuando los queramos cambiar tenemos que cambiarlos en dos lugares.

Si los escribimos una vez en cabecera.php y los incluimos con include, los cambiamos una vez y se actualizan en todas las páginas.

Primera regla de un sistema: no repetir código (DRY — Don't Repeat Yourself).

<?php include 'includes/cabecera.php'; ?>

<h1>Contenido propio de esta página</h1>

<?php include 'includes/pie.php'; ?>

7. Patrón mínimo de procesar.php, línea por línea 

<?php
  $nombre = $_POST['nombre'] ?? '';
  $dni    = $_POST['dni']    ?? '';

  if ($nombre == '' || $dni == '') {
     echo "Faltan datos";
     exit;
  }
?>
<!DOCTYPE html>
<html>
<body>
  <h1>Hola <?= $nombre ?>, tu inscripción fue registrada</h1>
</body>
</html>
$_POST['nombre'] ?? ''
  • $_POST es una variable especial de PHP que guarda los datos que llegaron por POST.
  • $_POST['nombre'] agarra el campo cuyo name="nombre" en el formulario.
  • ?? '' es el operador de coalescencia nula: "si esto no existe, usá esto otro".
  • Si el campo no vino, $nombre queda como '' y PHP no rompe.
¿Por qué el ???
Si escribís solo $_POST['nombre'] y el campo no vino, PHP tira un warning. El ?? evita ese warning.
if ($nombre == '' || $dni == '') { ... exit; }
  • Verifica que ningún campo crítico esté vacío.
  • Si lo está, muestra un mensaje y corta la ejecución con exit.
  • || significa "o": si alguno está vacío, se cumple la condición.
<?= $nombre ?>
  • Es la forma corta de <?php echo $nombre; ?>.
  • Se usa para imprimir una variable dentro del HTML.

8. Errores frecuentes

Cosas que pasan los primeros días con PHP. No es vergonzoso equivocarse: es vergonzoso seguir equivocándose después de que te lo explican.

El formulario no envía nada
  • Causa típica: falta method="POST" en el <form>, o falta name="..." en los inputs.
  • Cómo detectarlo: poné <?php var_dump($_POST); ?> al principio de procesar.php. Si imprime array(0) { }, no llegó nada.
action="" y no sé adónde apunta

action="" significa "enviar al mismo archivo". Mejor escribilo explícito: action="procesar.php".

Usé $_GET en vez de $_POST

Si el formulario usa method="POST", los datos están en $_POST. Si usa method="GET", en $_GET.

PHP es case-sensitive 
$_POST['Nombre']   // distinto de
$_POST['nombre']

Si el name del input es nombre, tenés que escribirlo igual en PHP.

Olvidar el ; al final de cada línea

PHP tira un Parse error. Revisar línea por línea.

No incluir cabecera.php ni pie.php

Sin include, terminás copiando el <head> y el <body> en cada archivo. Eso es repetir código.

10. Buenas prácticas

  1. Siempre validar lo que viene de $_POST.
  2. Usar include para no repetir cabecera y pie.
  3. Separar HTML de PHP cuando se pueda. Usar <?= ?>.
  4. Nombres con sentido. No usar $x, usar $nombre, $dniCliente.
  5. Indentar. Hace falta para leer el código después.
