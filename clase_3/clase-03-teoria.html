1. De if/elseif a switch

En la clase anterior armaste info.php con if y elseif para mostrar distinto contenido según el valor de $_GET['seccion']. Ese enfoque funciona, pero cuando tenés muchas secciones el código se vuelve difícil de leer.

Acá viene switch. Compará:

// Con if/elseif
$s = $_GET['seccion'] ?? 'inicio';

if ($s == 'inicio') {
    echo '<h2>Inicio</h2>';
} elseif ($s == 'ayuda') {
    echo '<h2>Ayuda</h2>';
} elseif ($s == 'contacto') {
    echo '<h2>Contacto</h2>';
} else {
    echo '<h2>404</h2>';
}
// Con switch
$s = $_GET['seccion'] ?? 'inicio';

switch ($s) {
    case 'inicio':
        echo '<h2>Inicio</h2>';
        break;
    case 'ayuda':
        echo '<h2>Ayuda</h2>';
        break;
    case 'contacto':
        echo '<h2>Contacto</h2>';
        break;
    default:
        echo '<h2>404</h2>';
}

Cómo funciona

  • switch evalúa la variable una sola vez y salta al case que coincide.
  • Cuando encuentra el case correcto, ejecuta el código hasta el break.
  • break sale del switch. Sin break, sigue ejecutando el siguiente case (se llama fall-through).
  • default es el "else" del switch: se ejecuta si ningún case coincide.

Fall-through: ¿para qué sirve?

A veces querés que dos valores compartan el mismo código:

switch ($rol) {
    case 'admin':
    case 'supervisor':
        echo 'Acceso completo';
        break;
    case 'editor':
        echo 'Puede editar contenidos';
        break;
    default:
        echo 'Acceso limitado';
}

Acá admin y supervisor hacen lo mismo porque el primer case no tiene break.

Cuidado: olvidar un break sin querer es el error más común con switch. Siempre revisá que cada case tenga su break (a menos que quieras fall-through a propósito).

2. Separando datos de la vista

Si escribís HTML adentro de cada case del switch, estás mezclando la lógica con la presentación. Para un par de secciones funciona, pero cuando el proyecto crezca se vuelve un desastre.

La idea es simple: el switch solo asigna valores a variables, y el HTML se escribe una sola vez abajo.

<?php
$seccion = $_GET['seccion'] ?? 'inicio';

// Valores por defecto
$titulo = 'Inicio';
$texto  = 'Bienvenido al sistema.';
$clase  = 'sec-inicio';

switch ($seccion) {
    case 'ayuda':
        $titulo = 'Ayuda';
        $texto  = 'Encontrá instrucciones y respuestas.';
        $clase  = 'sec-ayuda';
        break;
    case 'contacto':
        $titulo = 'Contacto';
        $texto  = 'Escribinos por cualquier consulta.';
        $clase  = 'sec-contacto';
        break;
}
?>
<!DOCTYPE html>
<html>
<body class="<?= $clase ?>">
  <h1><?= $titulo ?></h1>
  <p><?= $texto ?></p>
</body>
</html>

¿Qué ganamos?

  • Legibilidad: el switch tiene solo asignaciones, es fácil de leer.
  • Mantenibilidad: si querés cambiar cómo se ve la página, tocás el HTML de abajo, no el switch.
  • Escalabilidad: agregar una sección nueva es copiar un case con sus variables. No tocar nada del HTML.
Recordá: <?= $variable ?> es la abreviatura de <?php echo $variable; ?>. Pinta el valor directamente.

3. CSS dinámico por sección

Si cada sección tiene una clase distinta en el <body>, podés darle su propio estilo con CSS:

<!-- PHP elige la clase -->
<body class="<?= $clase ?>">

Y en el CSS:

/* Cada sección define su color de acento */
.sec-inicio   { --accent: #4a90d9; }  /* azul */
.sec-ayuda    { --accent: #e67e22; }  /* naranja */
.sec-contacto { --accent: #27ae60; }  /* verde */

/* El body usa ese color */
body { background: var(--accent); transition: background .3s; }

/* Los paneles pueden heredar el color */
.panel { border-left: 4px solid var(--accent); }

¿Qué podés cambiar por sección?

  • Color de fondo (como el ejemplo de arriba).
  • Imagen de fondo — background-image distinta por sección.
  • Borde del panel activo en el menú.
  • Color del navbar o del botón de la sección activa.
  • Transiciones entre secciones.
Tip: usá transition en el body para que el cambio de color sea suave. Con background: var(--accent); transition: background .3s; alcanza.

4. Mostrando imágenes

El mismo patrón de variables sirve para imágenes. Agregá $imagen al switch y mostrala en el HTML:

// Adentro del switch
case 'ayuda':
    $titulo = 'Ayuda';
    $texto  = 'Instrucciones de uso del sistema.';
    $imagen = 'img/ayuda.jpg';
    $clase  = 'sec-ayuda';
    break;

// En el HTML
<img src="<?= $imagen ?>" alt="<?= $titulo ?>">

Imágenes placeholder para desarrollo

Si no tenés las imágenes todavía, usá picsum.photos para generar placeholders:

$imagen = 'https://picsum.photos/seed/ayuda/600/300';
//         ──────────────┬─────────┬────┬───┬───
//                      seed      tema ancho alto
//         misma seed → misma imagen siempre

El parámetro seed hace que la imagen sea siempre la misma para esa sección. Después las reemplazás por imágenes reales.

Importante: las imágenes de placeholder son para desarrollo. Cuando el proyecto esté listo, las imágenes deberían ser propias o libres de derechos.

5. Git: primeros pasos

Git es un sistema de control de versiones. Guarda el historial de cambios de tu proyecto y te permite volver atrás si algo sale mal.

El flujo básico

Editarcambiás archivos
→
git addpreparás los archivos
→
git commitsacás la foto
→
git pushsubís la foto a GitHub

Paso a paso

1. Configurar Git (la primera vez):

git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"

2. Ir a la carpeta de tu proyecto y activar Git:

cd C:/xampp/htdocs/clase02/
git init

git init crea un repositorio vacío en esa carpeta. Se hace una sola vez por proyecto.

3. Agregar archivos y hacer el primer commit:

git add .
git commit -m "Primer commit: sistema con login y switch"
  • git add . agrega todos los archivos de la carpeta al "escritorio" (staging).
  • git commit -m "mensaje" saca la foto. El mensaje describe qué cambió.

4. Crear un repositorio en GitHub y subir:

  1. Entrá a github.com e iniciá sesión.
  2. Botón verde "New repository" (arriba a la derecha).
  3. Nombre: clase02 (o el que quieras). Dejá todo lo demás como está.
  4. Copiá estas líneas y ejecutalas en tu terminal:
git remote add origin https://github.com/tuusuario/clase02.git
git branch -M main
git push -u origin main

5. Verificá: entrá a github.com/tuusuario/clase02 — tenés que ver todos tus archivos.

Segundo commit (y siguientes)

Hacé un cambio, guardalo, y repetí:

git add .
git commit -m "Agrego seccion de ayuda a info.php"
git push

Cada commit es un punto de guardado. Si después algo se rompe, podés volver a cualquier commit anterior.

6. Errores frecuentes

Switch

Error Por qué pasa Solución
Se ejecutan dos cases Falta break en el primer case Agregar break; al final de cada case
Sale todo en blanco El ?> del switch no cierra o hay un error de sintaxis Revisar que todos los case terminen en :
No encuentra la sección El valor de $_GET no coincide exactamente Usar default para atrapar valores inesperados

Git

Error Por qué pasa Solución
fatal: not a git repository No ejecutaste git init en esa carpeta git init
GitHub pide usuario y contraseña y no acepta GitHub ya no acepta contraseñas; necesita token o GitHub CLI Usar gh auth login o generar token en Settings → Developer settings → Personal access tokens
failed to push porque hay commits remotos Alguien (o vos desde otra máquina) subió commits que no tenés git pull --rebase y después git push
Subí una carpeta equivocada Hiciste git init en el lugar incorrecto Borrar la carpeta .git y empezar de nuevo

7. Glosario

TérminoSignificado
switchEstructura de control que evalúa una variable y ejecuta el case que coincide.
breakInstrucción que sale del switch. Sin break, el código sigue al próximo case.
defaultCaso opcional del switch que se ejecuta si ningún case coincide.
Fall-throughComportamiento de switch cuando un case no tiene break: sigue ejecutando el siguiente.
var(--variable)Variable CSS. Permite definir un valor y reusarlo en varias reglas.
RepositorioCarpeta de proyecto con Git activado. Guarda todo el historial.
Commit"Foto" del estado actual de los archivos en un momento dado.
StagingÁrea donde preparás los archivos antes del commit (git add).
git initComando que activa Git en una carpeta (crea el repositorio).
git addAgrega archivos al staging para el próximo commit.
git commitGuarda un commit con los archivos que están en staging.
git pushSube los commits locales a GitHub.
GitHubServicio en la nube que almacena repositorios Git. Como un "drive" para tu código.

8. Buenas prácticas

  • Siempre break al final de cada case, salvo que quieras fall-through a propósito.
  • default siempre por si alguien escribe una URL inválida.
  • Switch solo asigna variables. El HTML se escribe abajo, separado.
  • Mensajes de commit descriptivos — "arreglo bug" no sirve. "Corrijo redirección después del login" es mejor.
  • Commit chicos, commit seguido. Un commit por cambio lógico. No esperes a tener todo el proyecto listo.
  • Editar → add → commit → push en ese orden. Nunca saltees pasos.
  • Mirá siempre git status antes de commitear para saber qué vas a incluir.
