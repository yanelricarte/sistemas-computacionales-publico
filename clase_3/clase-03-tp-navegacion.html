Navegación visual con switch y Git

Refactorizar info.php a switch con contenido en variables, imágenes, CSS dinámico por sección, y subir todo a GitHub.

Materia: Proyecto, Diseño e Implementación de Sistemas Computacionales · 7.º año
Tiempo de trabajo en clase: 30 minutos (Actividad 1) + 60 minutos (Git)

Objetivo

Transformar el info.php que ya tenés (con if/elseif) en una versión con switch, variables separadas, imágenes y CSS que cambia según la sección. Además, subir el proyecto completo a GitHub.

Consigna

Parte 1 — Refactorizar info.php a switch

  1. Abrí tu info.php de la clase anterior.
  2. Reemplazá los if/elseif por un switch.
  3. Cada case debe asignar variables ($titulo, $texto, $clase) en lugar de escribir HTML.
  4. El HTML (con <?= $variable ?>) va abajo del switch, una sola vez.
  5. Mantené al menos tres secciones: inicio, ayuda, contacto + una más inventada por vos.

Parte 2 — Imágenes

  1. Agregá una variable $imagen a cada case del switch.
  2. Mostrá la imagen en el HTML con <img src="<?= $imagen ?>">.
  3. Usá picsum.photos para las imágenes placeholder: 
    https://picsum.photos/seed/ayuda/600/300
    Cambiá seed/ayuda por seed/inicio, seed/contacto, etc. así cada sección tiene su imagen.

Parte 3 — CSS dinámico

  1. En el <body>, usá class="<?= $clase ?>".
  2. En tu css/estilos.css, agregá reglas para cada sección: 
    .sec-inicio   { --accent: #4a90d9; }
.sec-ayuda    { --accent: #e67e22; }
.sec-contacto { --accent: #27ae60; }
body { background: var(--accent); transition: background .3s; }
  3. Dale a cada sección un color distinto (fondo, acento, lo que se te ocurra).
  4. Opcional: agregá una transición suave al cambiar de sección.

Parte 4 — Subir a GitHub

  1. Abrí la terminal en tu carpeta del proyecto.
  2. Configurá Git (si no lo hiciste antes): 
    git config --global user.name "Tu Nombre"
git config --global user.email "tu@email.com"
  3. Inicializá el repositorio: 
    git init
  4. Agregá todos los archivos y hacé el primer commit: 
    git add .
git commit -m "Primer commit: sistema con login, switch y navegacion"
  5. Entrá a github.com, creá un repo nuevo (nombre: clase02 o el que quieras).
  6. Conectá y subí: 
    git remote add origin https://github.com/tuusuario/clase02.git
git branch -M main
git push -u origin main
  7. Verificá que los archivos estén en GitHub entrando a la URL de tu repo.

Parte 5 — Segundo commit

  1. Agregá UNA sección más a tu info.php (inventá un tema: "galería", "blog", "faq", etc.).
  2. Guardá los cambios y subilos: 
    git add .
git commit -m "Agrego seccion galeria a info.php"
git push
Opcional — techo: si terminás todo y querés más, agregá un navbar con CSS que marque la sección activa con un color distinto. Pista: usá la misma variable $clase para poner una clase en el link activo.

Errores comunes

ErrorQué revisar
La página muestra todo junto sin formato¿Cerraron bien el ?> antes del HTML?
El color de fondo no cambia¿Está class="<?= $clase ?>" en el <body>? ¿Las reglas CSS están bien escritas?
GitHub no acepta la contraseñaUsá un token de acceso personal (GitHub Settings → Developer settings → Personal access tokens) o instalá GitHub CLI y ejecutá gh auth login
Git dice "nothing to commit"¿Guardaste el archivo antes de hacer git add?
No encuentro las imágenesCon picsum.photos no necesitás descargar nada, están online.
